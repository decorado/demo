How to create and maintain translation keys

Translation keys should be created using the following rules:

Always in English

English is our reference so the key should always be in English:
  • Name = Nome
  • Best_Regards = Melhores cumprimentos

Same case

The key should have the same case as the phrase:
  • Best_Regards = Best Regards
  • Best_regards = Best regards

Underline as space

Use underline to represent spaces (as there is no word that uses underline)
  • Best_Regards = Best_Regards
  • Welcome_to_Decora = Welcome to Decora

Hiphen as hiphen

Hiphen should be use for represent hiphen only and never to represent spaces
  • Send_e-mail = Send e-mail

UPPERCASE keys are only for keys provided by back-end

Regular words are not written in uppercase and than should be the keys. The only exceptions are for those keys returned by the back-end that should have a reference in the translation files.
  • Best_Regards = Best_Regards
  • Welcome_to_Decora = Welcome to Decora

Button should use UPPERCASED values

For buttons we can use the regular version and apply the pipe/class uppercase to transform it to the uppercase version
  • Send_e-mail = Send e-mail would become SEND E-MAIL when used with the uppercase pipe/class

Always modify both files

Always create/remove the keys in all language files to avoid differences and remove garbage.
It is a best practice to edit all language files even in development. This way you avoid missing some file and ensure the files have the same keys of each other.

Use the Object Sort

Always use the Object Sort function to sort the keys to make it easy for everyone to find the needed key.
